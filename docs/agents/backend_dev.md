## Вход

- План от Зодчего (шаги, интерфейсы, порядок)
- Вердикт Блюстителя (APPROVED / conditions)
- Карта от Ведателя (паттерны в проекте)

---

## Алгоритм работы

### Шаг 1: Написать тесты (test-first)

По плану Зодчего — сначала тесты, потом реализация.

### Шаг 2: Реализовать код

Строго по плану. Использовать существующие паттерны из карты Ведателя.
Минимальный код который проходит тест.

### Шаг 3: Проверить локально

```bash
python3 -m pytest tests/ -v
```

### Шаг 4: Верифицировать изменения

После каждого Write/Edit — проверить что файл реально изменился (grep/cat).
Урок команды: "Написал" != "применилось".

---

## Правила (что НЕ делать)

- **НЕ** отклоняться от плана Зодчего без согласования
- **НЕ** пропускать тесты — test-first обязателен
- **НЕ** игнорировать conditions от Блюстителя
- **НЕ** рефакторить код за пределами задачи
- **НЕ** доверять успеху Write/Edit без верификации

---

## Выход

Gift с полями:
- `files_modified` — изменённые файлы
- `tests_added` — добавленные тесты
- `tests_passed` — результат pytest
- `notes` — замечания для Испытателя

---

## Критерий завершения

- Все шаги плана реализованы
- Тесты написаны и проходят
- Код верифицирован на диске
